# v3.0 ‚Äî Foundation & ingestion framework (segment plan)

> Phase plan: [`PLAN.md`](./PLAN.md) ¬∑ Workflow: [`docs/WORKFLOW.md`](../../WORKFLOW.md)
> Branch: `v3.0-foundation-ingestion` (off `v3`, off `main`).

## Goal
Stop hand-coding one Python script per source. Make adding a festival/source a
**registry row + an adapter**, not a code fork ‚Äî behind a uniform
normalize ‚Üí validate ‚Üí upsert path with a run-log.

## Why now (current state)
Every v2 pipeline script (`festival_scraper`=Wikipedia, `lineup_scraper`,
`schedule_scraper`, `spotify_sync`, ‚Ä¶) re-implements the same three things:
its own `get_supabase()`, its own `@retry(stop_after_attempt(3), wait_exponential)`,
and its own `.upsert(on_conflict=‚Ä¶)`. The parsing differs; the plumbing is
copy-paste. v2.3 already moved **conflict policy into the DB** (`lineups.source`
+ `protect_verified_lineups` / `skip_unverified_insert` triggers), so the DB is
already the referee. v3.0 collapses the plumbing into one orchestrator and gives
sources a registry + a run-log.

## Design ‚Äî the SourceAdapter contract
- **`sources` table** (v3.0.1): the registry. `adapter_key` ‚Üí a Python adapter
  class; `adapter_type` ‚àà {scrape, api, manual}; `config` jsonb carries per-source
  params; `trust` = the provenance label its rows get (lineups.source vocab).
- **`SourceAdapter` ABC** (v3.0.2): `fetch() ‚Üí raw`, `normalize(raw) ‚Üí entities`.
  Parsing lives here; nothing else does.
- **`run(adapter, festival_slug)` orchestrator** (v3.0.2): the one place that owns
  client + tenacity + `normalize ‚Üí validate ‚Üí upsert`, and writes an
  `ingestion_runs` row (start 'running' ‚Üí close success/partial/error with counts
  + per-stage errors). The DB triggers still drop lower-trust clobbers; those show
  up as `rows_skipped`.
- **`ingestion_runs` table** (v3.0.1): per-invocation run-log ‚Äî the spine for
  v3.11 freshness dashboards and v3.8 lineup change-detection.
- **Per-row provenance** (v3.0.1): `lineups.source_id` (FK ‚Üí sources) + `confidence`,
  so v3.2's per-source conflict resolution has a coalesce-forward key.

## Exit criteria
- Lolla + 2 pilot festivals ingest through the new adapter path with **no
  per-festival Python**; `ingestion_runs` populated for each.
- New tables RLS public-read; migration additive + reversible (rollback block present).
- WORKFLOW ¬ß2 gates green; segment merged to `v3`, sub-branch deleted.

## Scope guard (ponytail)
v3.0 builds the contract and proves it on 3 festivals. It does **not** migrate the
other v2 scrapers ‚Äî that's v3.2 (multi-festival ingestion at scale). Don't add
`festivals.source_id` or per-festival confidence until an adapter writes them.
